\begin{abstract}
Abstractive summarization is a sequence-to-sequence task.
A model reads a document and writes a shorter text.
Modern Large Language Model (LLM) checkpoints are good at generation and representation learning.
However, many of them do not provide a direct encoder-decoder interface for summarization.
Encoder-decoder models fit this task because the encoder reads the source and the decoder writes the summary.

This report presents LLM2Seq.
LLM2Seq reuses pretrained LLM source encoders inside an encoder-decoder summarizer.
A gated residual adapter maps source states into cross-attention memory.
A lightweight Transformer decoder then generates the summary.
We also study inference speed with self-distilled Multi-Token Prediction (MTP).
After the summarizer is trained, MTP heads learn from the frozen decoder.
They draft future tokens during decoding.
The main decoder verifies these drafted tokens.

We evaluate LLM2Seq on Vietnamese WikiLingua.
The results show that LLM-based source encoders can work as cross-attention memory for summarization.
They also show that MTP speedups should be measured by real throughput, not only by accepted tokens.
\end{abstract}

\noindent\textbf{Keywords:} abstractive summarization; encoder-decoder models; LLM source encoders; adapters; multi-token prediction; Vietnamese WikiLingua

\section{Introduction}
\label{sec:introduction}

\subsection{Motivation}

Abstractive summarization is rooted in a sequence-to-sequence framework.
The model must first represent the source document, then generate a concise target summary \citep{sutskever2014sequence,bahdanau2015neural,vaswani2017attention}.
Encoder-decoder architectures match this task structure by separating source comprehension from target generation.
In contrast, many modern LLMs use decoder-only architectures \citep{brown2020language,dubey2024llama3,yang2024qwen2}.
They are strong autoregressive generators, but they place the source context and target sequence inside one left-to-right causal stream.
This structural entanglement makes prompting simple, but it does not provide an explicit boundary between source representation, information filtering, and summary generation.

\subsection{Problem Statement}

This report investigates whether pretrained LLM layers and embedding models can be repurposed as source encoders inside an explicit encoder-decoder summarizer.
We study Vietnamese abstractive summarization on WikiLingua \citep{ladhak2020wikilingua}.
Given a source document $x$, the model must produce a salient summary $y$.
The goal is to reuse rich LLM source representations, preserve a clear cross-attention interface between encoding and decoding, and reduce decoder-side latency when possible.

\subsection{Gap with Existing Work}

Pretrained encoder-decoder models such as T5, BART, and mT5 remain strong paradigms for conditional text generation \citep{raffel2020t5,lewis2020bart,xue2021mt5}.
However, they do not directly reuse representations from recent LLM checkpoints.
Conversely, recent work shows that causal models can be converted into useful text encoders \citep{behnamghader2024llm2vec,luo2025lamate}.
Yet using these source states for summarization still requires an explicit and stable bridge to decoder cross-attention.

At the same time, end-to-end context compression methods such as LCLMs reduce inference cost by compressing long prompts into short latent sequences \citep{li2026contextcompression}.
This is useful for general long-context inference, but it is not the same objective as source-grounded summarization.
Abstractive summarization often depends on granular local evidence, including named entities, numbers, constraints, and ordered procedural steps.
If the document representation is compressed into a coarse latent bottleneck too early, the decoder can lose access to the local information needed to decide what belongs in the summary.
This leaves a gap for an architecture that can reuse LLM representations without sacrificing token-level source fidelity.
LLM2Seq addresses this gap by keeping token-level cross-attention memory, preserving fine-grained source states while explicitly separating the encoding and decoding pathways.

\subsection{Contributions}

We propose \textbf{LLM2Seq}, an encoder-decoder summarization framework built from repurposed LLM source encoders.
The contributions are:
\begin{itemize}
    \item We repurpose LLM-based source encoders for Vietnamese abstractive summarization, including a decoder-only-derived LLM2Vec encoder and a Qwen embedding encoder.
    \item We introduce a gated residual memory adapter that turns LLM hidden states into cross-attention memory for a lightweight decoder.
    \item We train self-distilled MTP heads after the summarizer is trained, keeping the main encoder-adapter-decoder path frozen \citep{gloeckle2024mtp,zhao2026mtpdistill}.
    \item We show that MTP must be evaluated by real throughput, because accepted tokens do not always become wall-clock speed.
\end{itemize}

\section{Background and Related Work}
\label{sec:related}

\subsection{Encoder-Decoder Summarization}

Encoder-decoder models separate reading from writing.
The encoder maps the input sequence into contextual states.
The decoder generates the output by attending to these states.
Attention-based sequence-to-sequence models introduced this interface \citep{sutskever2014sequence,bahdanau2015neural}, and the Transformer made self-attention and cross-attention standard \citep{vaswani2017attention}.
T5, BART, and mT5 show that this design works well for summarization \citep{raffel2020t5,lewis2020bart,xue2021mt5}.

\subsection{Decoder-only LLMs as Encoders}

Decoder-only LLMs are usually trained with next-token prediction.
Their causal mask is natural for generation but not for bidirectional source encoding.
LLM2Vec shows that decoder-only LLMs can be converted into strong text encoders \citep{behnamghader2024llm2vec}.
GRIT studies generative models as representation learners \citep{muennighoff2024grit}.
LaMaTE uses LLM encoders for machine translation \citep{luo2025lamate}.
T5Gemma explores encoder-decoder models built from modern decoder-only components \citep{zhang2025t5gemma2}.
LLM2Seq follows this line, but focuses on summarization.
The Llama variant is the cleaner decoder-only-derived encoder case, while the Qwen variant should be read as an embedding-oriented LLM source encoder.

\subsection{Context Compression and LCLM}

End-to-End Context Compression at Scale introduces LCLMs, which compress long contexts into shorter latent sequences for general long-context inference \citep{li2026contextcompression}.
LCLMs optimize compression ratio, memory, time-to-first-token, and long-context accuracy.
LLM2Seq uses an encoder-decoder memory interface, but it does not compress the source with a fixed ratio.
It keeps token-level memory because summaries often depend on local source evidence.

\subsection{Efficient Decoding and MTP}

Autoregressive decoding emits one token per step.
Speculative decoding drafts tokens and verifies them with a target model \citep{leviathan2023speculative}.
Blockwise decoding predicts several future tokens in parallel \citep{stern2018blockwise}.
Medusa adds extra decoding heads \citep{cai2024medusa}.
MTP methods train heads to predict future tokens directly \citep{gloeckle2024mtp}.
MTP-D adds self-distillation to improve these heads \citep{zhao2026mtpdistill}, following knowledge distillation \citep{hinton2015distilling}.

\subsection{Positioning of LLM2Seq}

LLM2Seq sits between encoder-decoder summarization and LLM source-encoder reuse.
It is not a general context compressor.
It is also not a pure decoder-only prompting method.
It asks a narrower question: can source-side LLM representations be adapted into useful cross-attention memory for summarization, and can verified MTP accelerate that decoder in practice \citep{gloeckle2024mtp,zhao2026mtpdistill}?
Table~\ref{tab:lclm-position} summarizes this position against end-to-end context compression \citep{li2026contextcompression}.

\section{LLM2Seq Method}
\label{sec:method}

\subsection{Problem Formulation}

Let $x=(x_1,\ldots,x_m)$ be the source document and $y=(y_1,\ldots,y_n)$ be the target summary.
Let $a^x_i\in\{0,1\}$ and $a^y_t\in\{0,1\}$ be the source and target masks.
The model estimates:
\begin{equation}
p(y|x)=\prod_{t=1}^{n}p(y_t \mid y_{<t},x).
\end{equation}
LLM2Seq makes the conditioning path explicit.
The source encoder builds token states:
\begin{equation}
H=E_{\theta}(x), \qquad H \in \mathbb{R}^{m \times d_e}.
\end{equation}
The adapter maps them into decoder memory:
\begin{equation}
M=A_{\phi}(H), \qquad M \in \mathbb{R}^{(m+g) \times d_d},
\end{equation}
where $g$ is the number of learned global memory tokens.
The decoder then generates:
\begin{equation}
p(y|x)=\prod_{t=1}^{n}p_{\psi}(y_t \mid y_{<t},M).
\end{equation}
Training uses teacher forcing.
With non-padding target positions $\Omega=\{t:a^y_t=1\}$, the summarization loss is:
\begin{equation}
\mathcal{L}_{sum}
=-\frac{1}{|\Omega|}\sum_{t\in\Omega}
\log p_{\psi}(y_t\mid y_{<t},M).
\end{equation}
This form is the core design choice.
The encoder handles source understanding.
The decoder handles target generation.
The adapter decides how source evidence is exposed to cross-attention.

\subsection{Overall Architecture}

Figure~\ref{fig:architecture} shows the full system.
LLM2Seq has three main parts.
First, a pretrained LLM-based source encoder is used; our instantiations are a decoder-only-derived LLM2Vec encoder and a Qwen embedding encoder \citep{behnamghader2024llm2vec,zhang2025qwen3embedding}.
Second, a gated residual memory adapter reshapes the encoder states into decoder memory.
Third, a lightweight Transformer decoder generates the summary with masked self-attention and cross-attention \citep{vaswani2017attention}.
After the summarizer is trained, an MTP module is attached for acceleration.
The MTP module is trained separately and verified by the main decoder during inference.

\subsection{Repurposed LLM Source Encoder}

We use two source encoders.
The Llama variant uses an LLM2Vec Sheared-LLaMA checkpoint \citep{behnamghader2024llm2vec}.
The Qwen variant uses Qwen3-Embedding-0.6B \citep{zhang2025qwen3embedding}.
This distinction matters for the scope of the claim: LLM2Seq studies reusable LLM source encoders, not only pure decoder-only checkpoints.
For a source sequence $x$, the encoder returns token-level hidden states:
\begin{equation}
H^{(\ell)}=F^{(\ell)}_{\theta}(x,a^x),
\qquad
H^{(\ell)}\in\mathbb{R}^{m\times d_e}.
\end{equation}
Unlike sentence embedding use cases, LLM2Seq keeps the full sequence of token states.
This is needed because the decoder must attend to local source evidence.
The encoder also returns a set of selected layers:
\begin{equation}
\mathcal{H}_S=\{H^{(l_1)},H^{(l_2)},\ldots,H^{(l_s)}\}.
\end{equation}
In our setting, the selected layers are $[-1,-4,-8,-12]$.
Full fine-tuning is expensive.
Phase 2 therefore applies LoRA to the encoder \citep{hu2022lora}.
For a frozen encoder weight matrix $W$, LoRA learns a low-rank update:
\begin{equation}
W'=W+\frac{\alpha}{r}BA,
\end{equation}
where $A\in\mathbb{R}^{r\times d_{in}}$ and
$B\in\mathbb{R}^{d_{out}\times r}$.
Only the low-rank matrices are updated for the encoder.
This adapts the source representation to Vietnamese summarization while keeping training practical.

\subsection{Gated Residual Memory Adapter}

The adapter is the bridge between the LLM encoder and the decoder.
It must solve a mismatch.
The encoder states live in the LLM representation space, while the decoder expects memory in its own hidden space.
The adapter also needs to keep token-level evidence, because summaries often depend on details in the source.

First, token-wise layer fusion combines selected encoder layers.
For token position $i$ and selected layer $j$, a score is computed by:
\begin{equation}
e_{ij}=w_f^\top H^{(l_j)}_i.
\end{equation}
The layer mixture is:
\begin{equation}
\alpha_{ij}
=\frac{\exp(e_{ij})}{\sum_{r=1}^{s}\exp(e_{ir})},
\qquad
\bar{H}_i=\sum_{j=1}^{s}\alpha_{ij}H^{(l_j)}_i.
\end{equation}
This lets different tokens use different depths of the LLM.
Lexical cues and semantic cues do not always appear at the same layer.

The fused states are then projected into decoder space.
The implemented gated residual projection first normalizes the fused state:
\begin{equation}
\hat{H}=\operatorname{LN}(\bar{H}).
\end{equation}
It builds a base path, a gate, and an update:
\begin{equation}
P=W_p\hat{H}+b_p,
\qquad
G=\sigma(W_g\hat{H}+b_g),
\end{equation}
\begin{equation}
U=W_2\operatorname{GELU}(W_1\operatorname{LN}(P)+b_1)+b_2.
\end{equation}
The adapted token state is:
\begin{equation}
R=P+G\odot U.
\end{equation}
The base path $P$ maps the encoder states into the decoder dimension.
The update block $U$ adds task-specific changes.
The gate $G$ controls how much of this update is used.
This residual form keeps the original source information easier to preserve.

A token salience gate then estimates which source positions should be emphasized:
\begin{equation}
S=\sigma(W_{s2}\operatorname{GELU}(W_{s1}\operatorname{LN}(R)+b_{s1})+b_{s2}).
\end{equation}
The implementation does not remove low-salience tokens.
It rescales them:
\begin{equation}
R'_i=(0.5+S_i)R_i.
\end{equation}
This lets the model give more weight to useful source tokens.
It also keeps the full source sequence for cross-attention.
Padding positions have $S_i=0$.
Thus padded memory cannot become salient.

An EncStack refines the memory with $L_a=4$ pre-norm Transformer encoder layers \citep{vaswani2017attention}:
\begin{equation}
\bar{R}^{(0)}=R',
\end{equation}
\begin{equation}
Q^{(u)}=\bar{R}^{(u-1)}
+\operatorname{SelfAttn}(\operatorname{LN}(\bar{R}^{(u-1)}),a^x),
\end{equation}
\begin{equation}
\bar{R}^{(u)}=Q^{(u)}
+\operatorname{FFN}(\operatorname{LN}(Q^{(u)})).
\end{equation}
Let $\tilde{R}=\bar{R}^{(L_a)}$.
The final memory concatenates learned global memory tokens $G_m$ with the refined token memory:
\begin{equation}
M=[G_m;\,\tilde{R}].
\end{equation}
The corresponding memory mask is:
\begin{equation}
a^M=[\mathbf{1}_g;\,a^x].
\end{equation}
The global tokens give the decoder a small document-level workspace.
The token memory gives the decoder direct access to local evidence.

\subsection{Lightweight Cross-Attentive Decoder}

The decoder is an 8-layer Transformer decoder with hidden size 1024, 16 attention heads, and a 4096-dimensional feed-forward layer \citep{vaswani2017attention}.
At step $t$, it reads the previous target tokens $y_{<t}$ and the adapted source memory $M$.
The decoder embeds target tokens with a tied input/output embedding matrix $E$:
\begin{equation}
Y^{(0)}=E[y_{<t}].
\end{equation}
It uses RoPE in causal self-attention \citep{su2021roformer}.
For a query-key pair $(q_i,k_j)$, RoPE rotates each head by position-dependent matrices:
\begin{equation}
\tilde{q}_i=R_iq_i,
\qquad
\tilde{k}_j=R_jk_j.
\end{equation}
Attention with mask $B$ is:
\begin{equation}
\operatorname{Attn}(Q,K,V,B)
=\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_h}}+B\right)V.
\end{equation}
The causal mask has $B_{ij}=-\infty$ for $j>i$.
The cross-attention mask uses $a^M$ so padded source tokens are ignored.

For decoder layer $r$, the pre-norm residual block is:
\begin{equation}
\tilde{Y}^{(r-1)}=\operatorname{RMSNorm}(Y^{(r-1)}),
\end{equation}
\begin{equation}
A^{(r)}=Y^{(r-1)}
+\operatorname{SelfAttn}(\tilde{Y}^{(r-1)},B_{causal}),
\end{equation}
\begin{equation}
\tilde{A}^{(r)}=\operatorname{RMSNorm}(A^{(r)}),
\end{equation}
\begin{equation}
C^{(r)}=A^{(r)}
+\operatorname{CrossAttn}(\tilde{A}^{(r)},M,a^M),
\end{equation}
\begin{equation}
Y^{(r)}=C^{(r)}
+\operatorname{SwiGLU}(\operatorname{RMSNorm}(C^{(r)})).
\end{equation}
The next-token distribution is:
\begin{equation}
o_t=E^\top\operatorname{RMSNorm}(Y^{(L_d)}_t),
\end{equation}
\begin{equation}
p_{\psi}(y_t\mid y_{<t},M)=\operatorname{softmax}(o_t).
\end{equation}
The decoder is intentionally smaller than the encoder.
This keeps generation cheaper while still allowing the encoder to carry strong source representations.

\subsection{Self-Distilled MTP}

After Phase 2, the main summarizer is frozen.
Phase 3 adds $K=4$ cascaded MTP heads \citep{gloeckle2024mtp,zhao2026mtpdistill,deepseekai2024deepseekv3}.
Let $h_t^{(0)}=Y^{(L_d)}_t$ be the frozen decoder state at target position $t$.
The main head predicts:
\begin{equation}
p_t^{(0)}=\operatorname{softmax}(E^\top h_t^{(0)}).
\end{equation}
Each MTP depth predicts one future token.
During training, depth $k$ receives the previous MTP hidden state and the teacher-forced embedding of $y_{t+k-1}$:
\begin{equation}
z_t^{(k)}
=\operatorname{LN}\left(W_c[h_t^{(k-1)};E(y_{t+k-1})]\right).
\end{equation}
Then a small causal block produces:
\begin{equation}
\bar{z}_t^{(k)}
=z_t^{(k)}+\operatorname{SelfAttn}(\operatorname{LN}(z_{\leq t}^{(k)})),
\end{equation}
\begin{equation}
h_t^{(k)}
=\bar{z}_t^{(k)}
+\operatorname{FFN}(\operatorname{LN}(\bar{z}_t^{(k)})).
\end{equation}
The MTP distribution is:
\begin{equation}
p_t^{(k)}=\operatorname{softmax}(E^\top h_t^{(k)}),
\qquad k=1,\ldots,K.
\end{equation}
Thus head $k$ predicts $y_{t+k}$.

The hard-label loss trains the heads to predict the gold future token:
\begin{equation}
\ell_k^{CE}
=-\frac{1}{|\Omega_k|}
\sum_{t\in\Omega_k}\log p_t^{(k)}(y_{t+k}),
\end{equation}
\begin{equation}
\mathcal{L}_{CE}
=\frac{1}{\sum_k w_k}
\sum_{k=1}^{K}w_k\ell_k^{CE},
\end{equation}
where $\Omega_k$ contains valid non-padding positions for offset $k$.
The self-distillation loss makes each MTP head follow the frozen main decoder at the same future position \citep{hinton2015distilling}.
Let $z_{t+k}^{(0)}$ be the detached main-head logits at position $t+k$.
We keep the top-$J$ vocabulary indices $\mathcal{V}_{t,k}$ from this teacher distribution.
With temperature $\tau$, the teacher and student distributions on this top-$J$ set are:
\begin{equation}
q_{t,k}=\operatorname{softmax}(z_{t+k,\mathcal{V}_{t,k}}^{(0)}/\tau),
\end{equation}
\begin{equation}
s_{t,k}=\operatorname{softmax}(z_{t,\mathcal{V}_{t,k}}^{(k)}/\tau).
\end{equation}
The KL loss is:
\begin{equation}
\mathcal{L}_{KL}
=\frac{\tau^2}{\sum_k w_k}
\sum_{k=1}^{K}w_k
\frac{1}{|\Omega_k|}
\sum_{t\in\Omega_k}\operatorname{KL}(q_{t,k}\,\|\,s_{t,k}).
\end{equation}
The final Phase 3 loss is:
\begin{equation}
\mathcal{L}_{MTP}
=\lambda_{CE}\mathcal{L}_{CE}
+\gamma(u)\lambda_{KL}\mathcal{L}_{KL},
\end{equation}
where $u$ is the training progress.
The ramp $\gamma(u)$ is 0 before 20\% of training, increases linearly until 50\%, and is 1 afterwards.
We use $\lambda_{CE}=0.6$, $\lambda_{KL}=0.2$, $\tau=1.5$, $J=2048$, and head weights of $[1.0, 0.8, 0.5, 0.25]$ following the self-distilled MTP setup.
Only the MTP heads are updated in this phase.

\subsection{Verified MTP Decoding Algorithm}

At inference time, the MTP heads draft future tokens.
The main decoder verifies them.
This keeps the output tied to the trained summarizer, following the draft-and-verify idea in speculative decoding \citep{leviathan2023speculative} and MTP-style future-token heads \citep{gloeckle2024mtp,zhao2026mtpdistill}.
Let $g(\pi)$ be the constrained greedy main-head token for prefix $\pi$.
It includes the same repetition penalty, minimum length rule, and no-repeat trigram constraint as standard decoding.
For a current prefix $\pi$, the main decoder first predicts:
\begin{equation}
c_0=g(\pi).
\end{equation}
The cascaded MTP module then drafts future tokens:
\begin{equation}
c_k=\operatorname*{arg\,max}_{v}
p^{(k)}(v\mid \pi,c_{<k}),
\end{equation}
\begin{equation}
k=1,\ldots,K.
\end{equation}
The candidate block is $c=(c_0,c_1,\ldots,c_K)$.
The verifier runs the frozen decoder on the candidate block and computes:
\begin{equation}
v_j=g(\pi,c_0,\ldots,c_j),
\qquad j=0,\ldots,K.
\end{equation}
The number of accepted draft tokens is:
\begin{equation}
a=\max\{r: c_j=v_{j-1}\ \text{for all}\ 1\leq j\leq r\}.
\end{equation}
The emitted block is:
\begin{equation}
b=(c_0,\ldots,c_a,v_a),
\end{equation}
unless an end token or maximum length cuts the block earlier.
This design is intended to follow the constrained main greedy path.
Accepted draft tokens reduce future decoding work, while $v_a$ corrects the first mismatch.

The full procedure follows verified draft-and-check decoding, which is summarized in Algorithm~\ref{alg:mtp_decoding}.

\begin{algorithm}[H]
\caption{Verified MTP Draft-and-Check Decoding}
\label{alg:mtp_decoding}
\begin{algorithmic}[1]
\State Encode the source once and cache cross-attention keys and values.
\Repeat
    \State Predict the main token $c_0$ from the current prefix.
    \State Draft $K$ future tokens with the cascaded MTP heads.
    \State Verify the candidate block with the frozen main decoder.
    \State Accept the longest draft prefix that matches the verifier.
    \State Emit the accepted prefix and the verifier correction token.
    \State Slice the decoder cache to the verified prefix.
\Until{the end token or the maximum length is reached}
\end{algorithmic}
\end{algorithm}
If step $s$ emits $e_s$ tokens, the average emitted tokens per step is:
\begin{equation}
\bar{e}=\frac{\sum_s e_s}{T},
\end{equation}
where $T$ is the number of verification steps.
A high $\bar{e}$ reduces the number of steps, but it does not guarantee higher throughput because verification also costs time.

\subsection{Training Procedure}

LLM2Seq uses three phases.
Phase 1 freezes the encoder and trains the adapter, decoder, global memory tokens, and LM head with $\mathcal{L}_{sum}$.
Phase 2 adds LoRA to the encoder and continues optimizing the summarization loss \citep{hu2022lora}.
Phase 3 freezes the main summarizer and trains only the MTP module with $\mathcal{L}_{MTP}$.
This ordering matters.
The summarizer is built first.
The acceleration module is trained after that, so it does not change the main summarization path.
The total trainable parameter set by phase is:
\begin{equation}
\Theta_1=\{\phi,\psi,E,G_m\},
\end{equation}
\begin{equation}
\Theta_2=\Theta_1\cup\{A_{LoRA},B_{LoRA}\},
\end{equation}
\begin{equation}
\Theta_3=\Theta_{MTP}.
\end{equation}

\subsection{Complexity and Inference Cost}

The encoder runs once per source document.
The autoregressive decoder then runs once per generated token.
Let $C_{\mathrm{AR}}$ be the average cost of one autoregressive decoding step.
If a summary has $n$ generated tokens, the decoding cost is roughly:
\begin{equation}
C_{\mathrm{decode}}^{AR}\approx nC_{\mathrm{AR}}.
\end{equation}
Verified MTP emits $\bar{e}$ tokens per step on average, but each step has extra draft and verification cost $C_{\mathrm{ver}}$:
\begin{equation}
C_{\mathrm{decode}}^{MTP}
\approx \frac{n}{\bar{e}}(C_{\mathrm{AR}}+C_{\mathrm{ver}}).
\end{equation}
MTP is faster only when:
\begin{equation}
\bar{e} > 1+\frac{C_{\mathrm{ver}}}{C_{\mathrm{AR}}}.
\end{equation}
This explains why accepted tokens and real throughput can disagree.
The system must save more decoder steps than it adds in verification overhead.

\subsection{Compared Systems and Baselines}

We compare three systems.
\textbf{LLM2Seq-Llama} uses the LLM2Vec-Sheared-LLaMA encoder \citep{behnamghader2024llm2vec}.
\textbf{LLM2Seq-Qwen} uses the Qwen3-Embedding-0.6B encoder \citep{zhang2025qwen3embedding}.
Both use the same adapter and lightweight decoder design.
\textbf{T5Gemma} is a T5Gemma-2-1B-1B LoRA baseline \citep{zhang2025t5gemma2,hu2022lora}.

\subsection{Implementation Details}

The source length is up to 3,072 tokens and the target length is up to 384 tokens during training.
Generation uses at most 256 new tokens.
All reported systems use greedy decoding and no-repeat trigram blocking.
LLM2Seq uses bf16/tf32 training.
Both LLM2Seq variants adapt the encoder with LoRA \citep{hu2022lora}.
The MTP module uses four heads and is trained after the main summarizer is frozen.
We use AdamW with betas $(0.9,0.95)$ and $\epsilon=10^{-8}$, a linear warmup, cosine decay, and gradient clipping at 1.0.
Phase 1 trains the adapter and decoder for 6 epochs with batch size 24, gradient accumulation 2, learning rate $2\cdot10^{-4}$, warmup ratio 0.03, and weight decay 0.01.
Phase 2 adapts the encoder with LoRA.
Phase 2 runs for 6 epochs with batch size 16, gradient accumulation 2, encoder learning rate $5\cdot10^{-4}$, decoder and adapter learning rate $5\cdot10^{-5}$, and weight decay 0.05. Both runs also use gradual unfreezing.
Phase 3 freezes the encoder, adapter, and decoder, then trains only the MTP heads for 6 epochs with batch size 20, gradient accumulation 4, learning rate $5\cdot10^{-4}$, warmup ratio 0.05, and weight decay 0.01.
Checkpoints are selected by validation loss.
For generation, all models use a minimum of 32 new tokens and repetition penalty 1.15.
T5Gemma is trained for 6 epochs with LoRA, batch size 16, gradient accumulation 2, learning rate $2\cdot10^{-4}$, warmup ratio 0.03, weight decay 0.01, cosine schedule, and fused AdamW.
Appendix~\ref{app:hparams} summarizes the training and decoding configuration.

\subsection{Evaluation Metrics}

We report ROUGE \citep{lin2004rouge}, BERTScore \citep{zhang2020bertscore}, length statistics, and speed.
ROUGE-1 and ROUGE-2 use unigram and bigram overlap.
Let $\hat{y}$ be the generated summary and $y$ be the reference.
For $n\in\{1,2\}$, let $C_n(\hat{y},y)$ be the clipped count of overlapping $n$-grams:
\begin{equation}
P_n=\frac{C_n(\hat{y},y)}{\#n\text{-grams}(\hat{y})},
\qquad
R_n=\frac{C_n(\hat{y},y)}{\#n\text{-grams}(y)}.
\end{equation}
The reported ROUGE-$n$ is F1:
\begin{equation}
\operatorname{ROUGE}\text{-}n
=\frac{2P_nR_n}{P_n+R_n}.
\end{equation}
ROUGE-L uses the longest common subsequence:
\begin{equation}
P_L=\frac{\operatorname{LCS}(\hat{y},y)}{|\hat{y}|},
\qquad
R_L=\frac{\operatorname{LCS}(\hat{y},y)}{|y|},
\end{equation}
\begin{equation}
\operatorname{ROUGE}\text{-}L
=\frac{2P_LR_L}{P_L+R_L}.
\end{equation}
For length, we report mean predicted words and too-long rate.
For Vietnamese text, ROUGE is computed with the Python \texttt{rouge-score} implementation using \texttt{use\_stemmer=False}.
Decoded strings are evaluated directly.
Length statistics use whitespace-separated tokens from \texttt{text.split()}, which we call words for readability.
We do not apply Vietnamese word segmentation such as VnCoreNLP or underthesea.
This choice is reported explicitly because Vietnamese word segmentation can affect n-gram overlap and length statistics.
BERTScore is computed between each generated summary and its reference using \texttt{bert-base-multilingual-cased}.
We report BERTScore F1 as a percentage.
BERTScore first encodes the generated summary and the reference into contextual token embeddings.
Let $\hat{E}$ be the generated-token embeddings and $E$ be the reference-token embeddings.
For each generated token, BERTScore finds the most similar reference token by cosine similarity:
\begin{equation}
P_B=\frac{1}{|\hat{E}|}\sum_{\hat{e}\in\hat{E}}\max_{e\in E}\cos(\hat{e},e).
\end{equation}
For each reference token, it also finds the most similar generated token:
\begin{equation}
R_B=\frac{1}{|E|}\sum_{e\in E}\max_{\hat{e}\in\hat{E}}\cos(e,\hat{e}).
\end{equation}
The reported BERTScore is the F1 score:
\begin{equation}
\operatorname{BERTScore}\text{-}F1
=\frac{2P_BR_B}{P_B+R_B}.
\end{equation}
BERTScore gives a semantic similarity signal that is less brittle than n-gram overlap, but it is still not a factuality or hallucination detector.
For speed, we report generated tokens per second and mean latency.
We compare speed only when the batch setting is the same.
When comparing AR and MTP speed, we use paired comparison logs under the same model and batch setting.

\subsection{Research Questions}

We evaluate LLM2Seq with three questions.
They cover summarization quality, encoder configuration, and decoding speed.
\begin{itemize}
    \item \textbf{RQ1}: Can pretrained LLM-based source encoders be reused inside an encoder-decoder summarizer?
    \item \textbf{RQ2}: How much do the LLM2Seq encoder configurations affect quality, length, and speed?
    \item \textbf{RQ3}: Does verified MTP improve real throughput, or does it only reduce decoding steps?
\end{itemize}

\section{Results and Analysis}
\label{sec:results}

\subsection{RQ1: Viability of LLM-based Source Encoders}

Table~\ref{tab:main-results} reports the main WikiLingua test results.
The Qwen-based LLM2Seq system achieves 54.01 ROUGE-1, 20.83 ROUGE-2, 31.83 ROUGE-L, and 73.42 BERTScore F1.
This shows that a pretrained LLM-based source encoder can provide useful cross-attention memory for an encoder-decoder summarizer.
The Llama-based system also produces valid summaries, but its lower ROUGE scores and much shorter outputs indicate that not every source encoder configuration transfers equally well.

Compared with T5Gemma, LLM2Seq-Qwen is close on ROUGE-1 but lower on ROUGE-2 and ROUGE-L.
This suggests that LLM2Seq is a viable method, but it does not outperform the T5Gemma baseline.
Overall, the Qwen-based result gives positive evidence for RQ1.
It shows that an LLM-based source encoder can support encoder-decoder summarization.
However, the result also depends on the encoder configuration and output length.

\subsection{RQ2: Effect of Encoder Configuration}

The two LLM2Seq variants show different quality and length behavior.
LLM2Seq-Llama under-generates, with an average output length of 29.10 words, far below the 51.86-word reference average.
LLM2Seq-Qwen generates longer and more complete summaries, with an average length of 58.31 words, which is closer to the reference.
This length difference partly explains the large ROUGE gap between the two LLM2Seq variants.

The comparison should be read as a controlled configuration-level comparison rather than a pure encoder-only ablation.
Both variants use the same overall LLM2Seq architecture, adapter design, decoder design, and decoding constraints.
However, the source encoder choice also changes the tokenizer, vocabulary, and representation space.

Table~\ref{tab:speed} shows that LLM2Seq-Llama is faster than LLM2Seq-Qwen in autoregressive decoding, with 114.74 tokens/s compared with 95.87 tokens/s.
A likely explanation is the decoder-side vocabulary size.
Because the decoder uses a tied input/output embedding matrix, the final LM head depends on the tokenizer vocabulary.
The larger Qwen vocabulary can make each decoding step more expensive, even though the Qwen-based system gives better summarization quality.
Therefore, RQ2 shows a trade-off: the Qwen configuration improves quality and length control, while the Llama configuration is faster but under-generates.

This suggests that accepted or emitted tokens alone are not enough to evaluate MTP.
The verifier also costs time.
Cache handling and tensor control flow also add overhead.
For this reason, we report both speed and quality for AR and MTP.
Although verified MTP is designed to stay constrained by the main decoder path, the current AR and MTP evaluation paths can still show small metric differences because they use separate cache handling, stopping behavior, and decoding-control code.
We therefore report quality scores for both modes instead of assuming exact output equivalence.

Figure~\ref{fig:mtp-latency} shows a derived latency accounting for the Qwen system.
Under an idealized assumption that each MTP decoding step has the same cost as an autoregressive step, emitting 2.30 tokens per step would reduce the mean latency from 0.779s to about 0.339s.
In practice, the observed MTP latency is 0.662s, leaving about 0.323s as residual overhead.
This is not kernel-level profiling.
It is a conservative accounting from the evaluation logs, and it suggests that verification, cache management, tensor control flow, and Python overhead are likely bottlenecks.

At the same time, the result is modest, so MTP speed depends on implementation details.

\subsection{Comparison with End-to-End Context Compression}

We do not claim a direct empirical comparison with LCLMs.
The current project does not include an LCLM baseline.
The comparison is conceptual.
As summarized in Table~\ref{tab:lclm-position}, LCLMs compress contexts into shorter latent sequences for general long-context inference \citep{li2026contextcompression}.
LLM2Seq keeps token-level source memory and trains for Vietnamese summarization.
This distinction matters because LLM2Seq does not optimize compression ratio; instead, it studies whether token-level LLM representations can serve as effective source memory for summarization.
A fair future comparison should add block compression baselines, such as mean-pooling encoder states by 4, 8, or 16 tokens before the decoder.

\subsection{Qualitative Example}

Appendix~\ref{app:qualitative} gives examples from the WikiLingua test prediction files.
It shows the same pattern as the aggregate results.
LLM2Seq-Llama is short but loses key content.
LLM2Seq-Qwen often keeps the broad topic and a concise style, but it can also make serious semantic errors.
T5Gemma is more fluent and covers more details, but it is also longer.
This example supports the main reading of Table~\ref{tab:main-results}: different LLM2Seq configurations lead to different content-selection and length behaviors.

\subsection{Error Analysis}

The results reveal three error patterns.
First, LLM2Seq-Llama under-generates, which hurts recall and leads to lower ROUGE.
Second, T5Gemma often generates much longer summaries.
This can improve overlap scores but reduces conciseness.
Third, verified MTP improves throughput only modestly even when it emits multiple tokens per step, showing that verifier overhead remains an important bottleneck.

\section{Discussion}
\label{sec:discussion}

\subsection{What LLM2Seq Demonstrates}

LLM2Seq shows that LLM-based source encoders can be reused inside an encoder-decoder summarizer.
The Qwen variant is close to T5Gemma on ROUGE-1 while producing shorter summaries.
This supports the main idea, but it does not show that LLM2Seq is better than standard encoder-decoder baselines.

\subsection{Why Source Encoder Configuration Matters}

The Qwen-based LLM2Seq variant achieves higher ROUGE than the Llama-based variant.
The high-level adapter and decoder design are the same.
However, the tokenizer, vocabulary, and output layers are different.
Thus, this is a controlled configuration-level comparison.
It is not a pure encoder-only ablation.
The adapter can only expose information that exists in the encoder states.

\subsection{Why MTP Speedup Is Limited}

For Qwen, MTP gives 2.30 tokens per step but only 1.17x real speedup.
This matches the cost condition in Section~\ref{sec:method}.
MTP helps only when accepted tokens pay for verification.

\subsection{When Token-Level Memory Is Better Than Compression}

Token-level memory is useful when the summary depends on local source evidence, which differs from fixed-ratio context compression \citep{li2026contextcompression}.
This is common in how-to articles, where small details can change the correct summary.
Compression may help when the source is very long and redundant.
The current results do not settle this trade-off.
They show that token-level memory is a reasonable design point for WikiLingua summarization.

\subsection{Practical Deployment Considerations}

The current verified decoder is an implementation-level prototype.
Future deployment should fuse verification, reduce Python control flow, and use optimized runtimes such as vLLM.
Quantization methods such as SmoothQuant may also help under memory limits \citep{xiao2023smoothquant}.

\section{Threats to Validity}
\label{sec:validity}

\subsection{Dataset Validity}

The experiments use only the Vietnamese subset of WikiLingua \citep{ladhak2020wikilingua}.
This dataset mainly contains WikiHow-style instructional articles, whose summaries often consist of short action-oriented steps.
As a result, the findings may not transfer directly to other summarization domains such as news, dialogue, scientific documents, legal documents, or long-form reports.
In particular, the usefulness of token-level source memory may differ when the source documents are more redundant, more factual, or more domain-specific than WikiLingua.

\subsection{Baseline Validity}

The report compares LLM2Seq against T5Gemma as a strong encoder-decoder baseline, but it does not include a broad set of summarization baselines such as mT5, viT5, mBART, or prompted decoder-only LLMs.
Therefore, the results should be interpreted as evidence that LLM2Seq is viable, not as evidence that it is state-of-the-art.
In addition, T5Gemma is compared for quality and length, but not for speed, because its available inference log uses a different batch setting from LLM2Seq.
A fully controlled comparison would require all systems to be evaluated under the same hardware, batch size, decoding constraints, and runtime implementation.

\subsection{Configuration Validity}

The Llama-based and Qwen-based LLM2Seq systems use the same high-level architecture, adapter design, decoder design, training schedule, and decoding constraints.
However, the source encoder choice also changes the tokenizer, vocabulary size, representation space, and tokenizer-dependent embedding/output layers.
Thus, the comparison is best understood as a controlled configuration-level comparison rather than a pure encoder-only ablation.
This limits how strongly we can attribute the observed differences to the encoder representations alone.

\subsection{Metric Validity}

The main automatic metrics are ROUGE and BERTScore.
ROUGE measures lexical overlap with the reference summary, while BERTScore adds a reference-based semantic similarity signal.
Neither metric fully captures factual consistency, fluency, coherence, or usefulness.
This matters because a model can obtain reasonable automatic scores while still producing wrong or unnatural phrases, as shown in the qualitative examples.
For Vietnamese, whitespace tokenization is also a source of variation because one semantic word can contain multiple whitespace-separated syllables.
Therefore, the reported ROUGE, BERTScore, and length statistics should be read together with qualitative inspection rather than as complete measures of summary quality.
Future work should add factuality-focused evaluation, such as human judgment or LLM-as-a-judge scoring for faithfulness to the source.

\subsection{Implementation Validity}

The verified MTP speed result depends on the current implementation.
Although MTP emits multiple tokens per decoding step, the verifier, cache management, tensor control flow, and Python overhead can reduce the actual wall-clock benefit.
A fused runtime or optimized inference engine may produce different speedups.
For this reason, the MTP result should be read as a result of our current implementation.
It is not a theoretical limit of verified MTP decoding.

\subsection{Generalization Validity}

The report studies two LLM2Seq encoder configurations and one summarization dataset.
More encoders, more languages, and more datasets are needed before making broader claims about LLM source encoders for summarization.
The current results support the feasibility of the approach, especially for the Qwen-based configuration, but they do not establish that the same design will work equally well across domains or languages.
Future work should evaluate the framework on additional Vietnamese and multilingual summarization benchmarks.

\section{Conclusion}
\label{sec:conclusion}

LLM2Seq studies how to build an encoder-decoder summarizer from repurposed LLM source encoders.
The model uses a gated residual adapter to map source-side LLM states into decoder memory.
It then uses a lightweight cross-attentive decoder to generate summaries.
On Vietnamese WikiLingua, the Qwen-based variant is competitive with T5Gemma on ROUGE-1 and produces shorter summaries.
Self-distilled MTP gives a real 1.17x speedup for Qwen.
Future work should add adapter ablations, add compression baselines, and optimize verified decoding.

\section*{Limitations}

This report does not include a full adapter ablation, so it cannot isolate the contribution of each adapter component such as layer fusion, salience gating, EncStack refinement, and global memory tokens.
It also does not include an LCLM-style block-compression baseline, so the comparison with context compression remains conceptual.
The evaluation relies on automatic metrics and does not include human factuality judgments.
Finally, the current demo is an engineering artifact for inspection, not evidence for user-facing reliability.

\section*{Ethics Statement}

This work uses an existing public summarization dataset and does not introduce human-subject data collection.
The system may still produce incomplete or incorrect summaries.
It should not be used as the only source of information in high-stakes settings.

\section*{Broader Impact}

LLM2Seq may make summarization systems easier to build from existing LLM checkpoints.
This can reduce training cost and make summarization more accessible for lower-resource languages.
At the same time, better summarization tools can also spread errors faster if users over-trust generated summaries.

\section{Architecture Details}

The adapter uses token-wise layer fusion, a gated residual projection, token salience gating, EncStack memory refinement, and learned global memory tokens.
The decoder attends to the final memory $M$ through cross-attention in every layer.

\section{Dataset Preprocessing Details}

WikiLingua examples contain source sentence lists and target sentence lists.
Preprocessing joins source sentences into one source document and joins target sentences into one summary paragraph.
One test record with an empty source is removed.
